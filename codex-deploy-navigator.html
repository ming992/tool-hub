三平台完整流程

路径 A：GitHub 仓库部署

整理项目文件夹

确认真正的项目根目录在哪里。看到 package.json 或 index.html 的那一层通常就是根目录。

本地先跑一遍

纯 HTML 直接打开；框架项目先运行 npm install 和 npm run build。

上传 GitHub

仓库名尽量简单：不要中文、不要空格，例如 codex-landing-page。

在 Vercel 导入项目

使用 GitHub 登录，点击 Import Project，选择对应仓库。

检查配置

重点看 Framework Preset、Build Command、Output Directory。Vite 通常是 npm run build + dist。

点击 Deploy

等构建完成，打开 Production 链接，在电脑和手机都测试一遍。

路径 B：Vercel CLI 部署

CLI 就是用命令行把项目发到 Vercel。小白可以先不用，但进阶时很有用。

npm i -g vercel vercel vercel --prod

npm i -g vercel：安装 Vercel 命令工具。
vercel：创建一次预览部署。
vercel --prod：发布正式线上版本。

小白建议：先用 GitHub + 页面导入跑通一次，再学命令行。不要一开始被终端报错吓退。

路径 A：拖拽部署，HTML 小白优先

找到要上传的文件夹

纯 HTML 上传包含 index.html 的文件夹；Vite/React/Vue 先构建，上传 dist 或 build。

打开 Netlify Drop

登录后进入拖拽上传区域，把文件夹拖进去。

等待发布

发布完成后会获得一个 netlify.app 链接。

测试页面

检查首页、按钮、图片、样式、手机端。如果坏了，修好后重新拖拽上传。

保存版本记录

把项目名、平台、链接、问题写到下面的部署记录区，下一次不会忘。

路径 B：GitHub 自动部署

Add new site

选择 Import an existing project，连接 GitHub。

选择仓库

选择你刚上传的项目仓库。

填写构建设置

Build command 常见为 npm run build，Publish directory 常见为 dist 或 build。

点击 Deploy

之后每次 push 代码，Netlify 会自动重新发布。

纯 HTML 如果没有构建命令，Build command 可以留空，Publish directory 指向包含 index.html 的目录。

路径 A：Git 集成部署

进入 Workers & Pages

创建 Pages 项目，选择连接 GitHub 或 GitLab 仓库。

选择项目仓库

确认分支，一般是 main 或 master。

配置构建

填写 Build command 和 Build output directory。Vite 通常是 npm run build 与 dist。

部署并测试

等待构建完成，打开 Pages 提供的链接，检查图片、样式、刷新路由。

路径 B：Direct Upload 直接上传

准备静态输出文件夹

必须是已经能直接上线的文件：HTML、CSS、JS、图片等。框架项目先构建。

创建 Direct Upload 项目

进入 Pages 的直接上传路径，上传文件夹。

拿到链接

打开链接测试。如果你后续要自动部署，建议下一个项目直接用 Git 集成。

注意：Cloudflare Pages 的 Git 集成和 Direct Upload 是不同创建路径。新建项目时先想清楚要哪种方式。

构建命令和输出目录助手

部署表单里最容易填错的两个位置：Build Command 和 Output / Publish Directory。

项目类型构建命令输出目录小白提醒

Codex 部署流程导航器
