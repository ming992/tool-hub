Mindset

Codex 小白认知地图

小白用 Codex 的重点不是“马上成为程序员”，而是学会把需求说清楚、把结果跑起来、把问题反馈回去。

🛠️ 它能帮你做什么

Codex 可以帮你生成网页、小工具、自动化脚本、页面交互、修复报错、解释代码、重构结构。你负责提出真实需求，它负责把需求翻译成可运行的代码。

🧱 它不是万能许愿机

一句“帮我做个网站”通常会得到平庸结果。你需要告诉它：给谁用、解决什么问题、有哪些模块、需要什么交互、视觉风格是什么。

📄 从 HTML 单页开始

HTML 单页产品最适合小白：打开浏览器就能运行，不需要后端，不需要数据库，容易修改，也适合做提示词工具、计划表、导航站和资料包。

🚫 不要一上来做复杂 App

不要刚开始就做登录、支付、数据库、多人协作和大型系统。先做一个小而完整的工具，跑通一次“需求 → 代码 → 运行 → 优化”。

📝 好提示词 = 小型需求文档

优秀提示词要包含角色、目标用户、核心功能、页面结构、交互规则、视觉风格、输出格式、验收标准和禁止事项。

🔁 核心能力是迭代

小白不需要立刻看懂所有代码，但要学会描述问题：哪个按钮不工作、哪个区域不好看、手机端哪里乱、想新增什么功能。

Workflow

Codex 对话工作流

把 Codex 当成“会写代码的执行伙伴”，而不是许愿池。你说得越清楚，结果越接近产品。

1

先说目标

我要做什么产品？解决谁的问题？用户打开后第一件事要做什么？

2

再说结构

页面包含哪些模块？每个模块展示什么内容？哪些区域需要按钮、输入框、结果区？

3

补充交互

按钮点击后发生什么？内容是否要保存？是否需要一键复制、折叠、筛选、打卡？

4

描述风格

不要只说“高级”，要说清楚：浅色、卡片、圆角、渐变、留白、移动端自适应。

5

要求完整输出

明确要求输出完整 HTML 文件，包含 CSS 和 JavaScript，不要只给片段。

6

运行后再迭代

截图或描述问题，让 Codex 局部修改，不要每次推倒重来。

错误示范

帮我做一个很好看的网页，要高级，要能赚钱。

问题：没有用户、没有功能、没有页面结构、没有输出格式，AI 只能猜。

正确示范

请为零基础宝妈做一个“家庭每周计划表生成器”HTML 单页。包含成员选择、任务输入、自动生成周计划、一键复制、打印样式、手机端适配。视觉风格要温暖清爽，输出完整 HTML。

优点：用户明确、场景明确、功能明确、交互明确、输出明确。

How to use

产品使用说明

按照这个顺序使用，不用纠结从哪里开始。

先做新手自测。明确自己是完全小白、提示词小白、页面优化小白，还是准备部署变现。
按七天路线打卡。每天只完成一个核心动作，不要同时学习太多工具。
从项目库选一个最简单的项目。建议先做计划表、提示词生成器、标题生成器、个人主页这类小工具。
复制提示词给 Codex。不要只复制一句话，把用户、模块、功能、风格和输出要求都说清楚。
把生成结果保存为 HTML。新建文本文件，保存为 .html 后缀，用浏览器打开。
运行、截图、反馈、迭代。把问题现象讲清楚，让 Codex 局部修复。
