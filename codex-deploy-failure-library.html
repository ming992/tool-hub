错误日志识别器

把 Vercel/Netlify/Cloudflare/GitHub Actions 的错误日志粘贴进来，系统会按关键词给出可能原因和下一步。

排查优先级

  1. 先复制日志中第一条真正报错，不要只看最后一行。
  2. 本地执行 npm install 和 npm run build。
  3. 确认 Build Command、Output Directory、Root/Base Directory。
  4. 确认环境变量在 Production 环境中存在。
  5. 部署成功后，用 Console + Network 排查白屏、404、接口失败。
快速判断

构建失败看平台日志；页面白屏看浏览器 Console；接口失败看 Network；域名失败看 DNS；移动端失败看 CSS 断点。
